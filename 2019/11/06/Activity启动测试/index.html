Activity启动测试

0x00 写在前面

前几天漏洞审核过程中，发现攻击者伪造的攻击页面可以直接通过Intent调起APP暴露的一些Activity。如果使用这种攻击方式，就能够将一些只有本地安装恶意APP才能成功的攻击方式，变成远程类似钓鱼之类的攻击。

因此很好奇是否所有被暴露的Activity都可以使用这种方式被调起来，于是学习了一下Scheme启动APP。顺便也整理了一下APP测试常用的Activity启动方式.

0x04 Activity测试中常用的启动方式

除了上面讲的这种方式来启动暴露的Activity以外，我们还可以用一下的方式来启动

0x04-1 ADB am

adb中的am命令可以启动暴露的Activity，Service，Broadcast。

显式启动Activity ：adb shell am start -n ${your_package_name}/${your_activity_name}

隐式启动Activity : adb shell am start -a action,-d date,-m MIME-TYPE,-c category,-e 扩展数据

发送广播 : adb shell am broadcast -a ${broad_action_name}

启动服务：adb shell am startservice -n ${your_package_name}/${your_service_name}
